## 🎯 Resumo executivo

Esta release entrega duas frentes coordenadas:

1. **`mintly-lib` v1.1.0** — troca o motor de validação de dados interno da Mintly por uma biblioteca dedicada e versionada (o **Sapphire**, publicada oficialmente no nosso registro de pacotes).
2. **`MintlyApi`** — consome a nova versão da lib e ganha **três melhorias de fundação** na sua base técnica (contexto de requisição, tipagem das rotas e consultas ao banco mais seguras).

Na prática, deixamos de manter "no braço" um componente caseiro de validação e passamos a consumir uma peça reutilizável, testada e com versionamento próprio — e, na API, reforçamos a base sobre a qual todas as funcionalidades futuras serão construídas. **Não há mudança de funcionalidade visível para o usuário final** — é uma melhoria de fundação técnica que reduz custo de manutenção e abre caminho para validação de dados direto no banco.

---

## 💡 Por que isso importa para o produto

| Antes | Depois |
|-------|--------|
| Validador interno ("aurora") mantido dentro do próprio projeto | Biblioteca **Sapphire** dedicada, versionada e reutilizável entre projetos |
| ~660 linhas de código próprio para manter | Código removido; manutenção centralizada na lib externa |
| Tipo de dados e validação definidos separadamente (risco de divergir) | **Uma única fonte de verdade**: o tipo é derivado automaticamente da validação |
| API com "atalhos" técnicos na base (rotas sem tipagem, consultas que "adivinhavam" o formato) | Base da API explícita, tipada e preparada para crescer |

**Benefícios:**
- ✅ **Menos código para manter** → menos bugs e menor custo de evolução.
- ✅ **Consistência garantida** entre o que validamos e o que o sistema entende como dado válido.
- ✅ **Reaproveitamento** — a mesma base de validação pode ser usada por outros serviços da Mintly.
- ✅ **Preparação para o futuro** — habilita validar dados diretamente no banco (MongoDB) e adicionar rastreabilidade de requisições (usuário, trace) sem retrabalho.

## 📋 O que foi entregue

### Biblioteca — `mintly-lib` v1.1.0
1. Migração completa da validação interna para a biblioteca **`@ascendance-hub/sapphire-core`** (Sapphire).
2. Remoção do componente interno antigo ("aurora"), eliminando código duplicado.
3. Modelo de dados de **Pessoa** reescrito sobre a nova base, com tipagem derivada automaticamente do schema.
4. Atualização de versão: **1.0.1 → 1.1.0**.

### API — `MintlyApi` (release `release-20260602`)
1. **Migração para o Sapphire** acompanhando a lib (mesma troca de validação na camada da API).
2. **Contexto de requisição (`RequestContext`)** — em vez de passar só o "ambiente" como texto solto entre as camadas, a API agora carrega um objeto de contexto explícito da entrada da requisição até o banco. Isso abre espaço para anexar dados como usuário e rastreamento (trace) no futuro **sem reescrever o sistema todo**.
3. **Tipagem das rotas** — todas as rotas da API passaram a ser fortemente tipadas (entrada, parâmetros e cabeçalhos), eliminando "pontos cegos" onde erros só apareceriam em produção.
4. **Consultas ao banco mais seguras** — o mecanismo de consulta deixou de "adivinhar" o formato recebido e passou a exigir um tipo explícito, com erro claro (`501 Not Implemented`) quando um banco não suporta determinada consulta. Prepara o terreno para um futuro backend SQL sem quebrar o que existe.
5. A **versão da API permanece `1.0.0`** — é um serviço deploável (não um pacote publicado), então o marco é registrado pela tag/release **`release-20260602`** no GitHub.
